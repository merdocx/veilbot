## [2.4.20] - 2026-01-17

### Изменено
- ✅ **Изменена логика расчета expires_at при продлении подписки**:
  - При создании новой подписки - пересчет на основе всех платежей (как было)
  - При продлении существующей подписки - простое продление: текущий expires_at + длительность нового платежа
  - Это более интуитивно и соответствует ожиданиям пользователей

### Добавлено
- ✅ **Retry механизм для создания ключей при таймаутах**:
  - До 3 попыток создания ключа на каждом сервере при таймауте
  - Задержка 2 секунды между попытками
  - Retry только для таймаутов (не для других ошибок)
  - Закрытие клиента перед следующей попыткой для чистого переподключения
  - Решает проблему частых пропусков создания ключей на сервере #19 из-за таймаутов

### Semantic Versioning
- **MAJOR** - Несовместимые изменения API
- **MINOR** - Новая функциональность (обратно совместимо)
- **PATCH** - Исправления ошибок (обратно совместимо)

- 🆕 **Добавлено** - Новая функциональность

- 🐛 **Исправлено** - Исправления ошибок
- 🗑️ **Удалено** - Удаление функциональности
- 🔒 **Безопасность** - Улучшения безопасности
