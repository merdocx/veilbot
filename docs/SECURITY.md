# Политика безопасности VeilBot

## 🔒 Обзор безопасности

VeilBot разработан с учетом современных стандартов безопасности для защиты пользователей, данных и инфраструктуры. Этот документ описывает меры безопасности, уязвимости и процесс их устранения.

## 🛡️ Меры безопасности

### **Аутентификация и авторизация**

#### **Telegram Bot**
- ✅ Валидация Telegram Bot Token
- ✅ Проверка подлинности пользователей через Telegram API
- ✅ Ограничение доступа к административным функциям

#### **Веб-админка**
- ✅ Хеширование паролей с использованием bcrypt
- ✅ Защита от брутфорс атак (rate limiting)
- ✅ CSRF токены для защиты от межсайтовых запросов
- ✅ Сессии с ограниченным временем жизни
- ✅ Принудительное использование HTTPS

### **Защита данных**

#### **База данных**
- ✅ Шифрование конфиденциальных данных
- ✅ Подготовленные запросы для предотвращения SQL-инъекций
- ✅ Валидация всех входных данных
- ✅ Ограничение доступа к файлам БД

#### **Конфигурация**
- ✅ Переменные окружения для секретов
- ✅ Отдельные конфигурации для разных окружений
- ✅ Валидация конфигурации при запуске
- ✅ Безопасное хранение ключей шифрования

### **Сетевая безопасность**

#### **HTTP заголовки**
```python
SECURITY_HEADERS = {
    "X-Content-Type-Options": "nosniff",
    "X-Frame-Options": "DENY",
    "X-XSS-Protection": "1; mode=block",
    "Referrer-Policy": "strict-origin-when-cross-origin"
}
```

#### **Rate Limiting**
- 🔒 Логин: 5 попыток в минуту
- 🔒 API запросы: 100 запросов в минуту
- 🔒 Платежи: 5 попыток за 5 минут
- 🔒 Создание ключей: 10 запросов за 5 минут

### **Логирование безопасности**

#### **Security Logger**
- 📝 Детальное логирование всех действий
- 📝 Оценка рисков для подозрительной активности
- 📝 Отслеживание неудачных попыток
- 📝 Алерты при критических событиях

#### **Типы событий**
- 🔐 Попытки аутентификации
- 💳 Платежные операции
- 🔑 Создание/удаление ключей
- 👤 Административные действия
- ⚠️ Подозрительная активность

## 🔍 Мониторинг безопасности

### **Автоматический мониторинг**

#### **Security Logger**
```python
# Пример использования
from security_logger import log_payment_attempt

log_payment_attempt(
    user_id=12345,
    amount=500,
    protocol="outline",
    success=True
)
```

#### **Метрики безопасности**
- 📊 Количество неудачных попыток входа
- 📊 Подозрительная активность
- 📊 Попытки обхода rate limiting
- 📊 Аномальные паттерны использования

### **Ручной мониторинг**

#### **Проверка логов**
```bash
# Логи безопасности
tail -f /var/log/veilbot/veilbot_security.log

# Логи бота
./manage_services.sh logs
# или
sudo journalctl -u veilbot.service -f

# Логи админки
./manage_services.sh admin-logs
# или
sudo journalctl -u veilbot-admin.service -f
```

#### **Проверка состояния**
```bash
# Статус сервисов
./manage_services.sh status
# или
sudo systemctl status veilbot.service veilbot-admin.service

# Проверка процессов
ps aux | grep -E "(bot\.py|admin\.main)"
```

## 🔐 Секреты в истории Git

Если в репозиторий когда-либо попали пароли, SSH-ключи или токены (в том числе в удалённых файлах вроде `copy_to_new_server.py`):

1. **Считайте секрет скомпрометированным** — смените пароль или ключ на сервисе/сервере.
2. Удаление файла в новом коммите **не стирает** его из старых коммитов; при необходимости полной очистки истории используйте [git-filter-repo](https://github.com/newren/git-filter-repo) или BFG, затем согласованный `git push --force` (все клоны пересинхронизировать).
3. Не коммитьте секреты в открытый текст; используйте переменные окружения и `.gitignore` для локальных бэкапов (`venv.backup*/` и т.п.).
